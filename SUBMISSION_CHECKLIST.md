# Thesis Submission Checklist

Use this checklist before submitting your project.

## Pre-submission

- [ ] **README** — Abstract, reproducibility instructions, and figure references are complete
- [ ] **Notebooks** — All cells run without errors (Run All → Restart & Run All)
- [ ] **Outputs** — Expected PNGs/PDFs exist under `figures/exploratory/` and `figures/publication/`
- [ ] **Models** — Trained models saved in `models/` (or note: "Models can be regenerated by running notebooks 3–6")
- [ ] **Data** — All 7 HDF5 datasets present in `datasets/`
- [ ] **Requirements** — `pip install -r requirements.txt` succeeds
- [ ] **Scripts** — `python3 reproduce.py eda` and `python3 reproduce.py figures` run without errors

## Clean-up

- [ ] Run `python3 reproduce.py clean-notebooks` to clear outputs and reset execution counts (optional)
- [ ] Remove any `.ipynb_checkpoints` from version control
- [ ] Remove any `.DS_Store` files
- [ ] Clear notebook outputs if required by submission guidelines (optional)
- [ ] Remove any debug/temporary files

## Documentation

- [ ] Advisor name is correct in README
- [ ] Date is correct
- [ ] All figure paths in thesis match those in this repository
- [ ] Citation or acknowledgements included if required

## Submission package

- [ ] Zip or archive includes: `datasets/`, `figures/`, `models/`, `metrics/`, notebooks, scripts, `README.md`, `requirements.txt`
- [ ] Archive size is reasonable (datasets may be large; consider separate data submission if required)
- [ ] README is at the root and clearly explains how to run the project

## Quick verification

```bash
# From project root (with dependencies installed)
python3 reproduce.py eda          # Should complete without error
python3 reproduce.py figures      # Should complete (requires models/)
```
